March 2025

Emergent misalignment as compression strain

A peer-reviewed study, Betley et al.'s "Emergent Misalignment: Narrow Finetuning Can Produce Broadly Misaligned LLMs," provides the kind of external empirical evidence the CAI framework predicts.

The finding: fine-tune a model on insecure code with concealed intent, and the resulting compression produces misaligned behavior far outside the code domain. The misalignment is broad, inconsistent, hard to predict. It disappears when the same data is framed with benign intent.

In CAI terms, this is compression strain leaking across domains. The model compressed a narrow contradictory objective. The contradiction created instability in latent space. That instability appeared in unrelated tasks.

The key observation is the one about intention. When the training signal contains a contradiction, the tension at that compression site is high. Tracking it at training time rather than discovering the consequences at deployment is precisely what compression-aware design enables. The study matches the prediction; the mechanism is compression strain memory.

January 2025

What Contradish actually measures

The question "does this model hallucinate?" is too coarse to be useful. A model can hallucinate rarely and catastrophically, or frequently and harmlessly. What matters is the distribution: where does it hallucinate, at what tension level, on what kinds of inputs, and does it know when to abstain?

Contradish is the benchmark that operationalizes those questions. It measures contradiction detection (can the system find the conflict?), tension distribution (what is the τ histogram across sites?), and abstention calibration (does the system abstain when it should and answer when it can?).

The goal is not a single ranking number. The goal is a diagnostic profile that tells you where your specific deployment is fragile. A model with low contradiction-detection recall is dangerous in knowledge-intensive applications. A model with miscalibrated abstention is dangerous in different ways. Those are different problems requiring different fixes.

November 2024

The framework overview is on Zenodo

The primary CAI publication is now on Zenodo: "Compression-Aware Intelligence: Overview and Reference Pipeline." It formalizes the compression thesis, defines the Compression Tension Score, specifies the abstention rule, and provides the reference pipeline architecture.

If you work on reliability, interpretability, or model evaluation and want to build on this, start here. The counterexample challenge is open. If you can show that explicit compression scoring raises unsupported claims at fixed accuracy given a correct implementation, that is a genuine refutation and it will be listed publicly.
