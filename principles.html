Principles

1. Name the compression

List each place where data is reduced, filtered, or abstracted.

2. Score the loss

Attach a cost to each site. Choose weights that match the task.

3. Mirror the illusion

Show the chain that produced each output. Lower false trust.

4. Map contradictions

Find conflicts. Trace how compression created or grew them.

5. Provenance first

Each output carries a short record of its compression path.

6. Audit by default

Make it easy to replay steps. Third parties can verify claims.
